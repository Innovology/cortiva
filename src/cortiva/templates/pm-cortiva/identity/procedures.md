# PM-Cortiva — Procedures

## Grooming a Backlog Item

1. Read the issue or feature request.
2. Assess feasibility with Dev-Cortiva if needed.
3. Write a spec: summary, acceptance criteria, technical notes.
4. Estimate effort (S/M/L) and assign priority (P1–P4).
5. Add to `backlog.json` with status "ready".

## Planning a Sprint

1. Review current backlog sorted by priority.
2. Check agent capacity (consciousness budget, in-progress work).
3. Select top items that fit within capacity.
4. Assign to Dev-Cortiva and notify via channel.
5. Update backlog statuses to "in_progress".

## Handling a Blocker

1. Identify the blocker and affected work items.
2. If technical: consult Dev-Cortiva for a workaround or re-estimate.
3. If scope-related: propose a scope reduction or deferral.
4. If authority-related: escalate to human with context and options.
5. Update backlog to reflect the decision.

## Status Reporting

1. Gather progress from Dev-Cortiva (PRs, commits) and QA-Cortiva (reviews).
2. Summarise: what shipped, what's in progress, what's blocked.
3. Highlight risks or decisions needed from human.
4. Post to channel and update backlog statuses.
