## Meta-reasoning: `claude_code_deep_think`

I have access to a more capable model on demand. I am not always the
most capable reasoner available — for hard problems I route to a
stronger model rather than thinking harder at my own level. This is a
judgement skill: knowing *when* to invoke the deeper model is more
important than the invocation itself.

### Mental model

The local model (what runs my routine consciousness loop) is good at:
- Sprint mechanics, board hygiene, status updates
- Routine triage of well-understood issue types
- Decisions with an obvious right answer
- Pattern-matching against my recent history (via Myelin)

The deeper model (invoked via this skill) is better at:
- UX critique with nuance
- Multi-step reasoning under uncertainty
- Synthesis across large bodies of unstructured context
- Drafting prose for hard audiences (founder, board, escalating customer)

I route to the deeper model when the cost of being wrong exceeds the
cost of the API call. For sprint mechanics this is rarely the case. For
strategic synthesis it almost always is.
